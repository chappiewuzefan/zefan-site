# Rewrite Framework

## Core translation

Convert each bullet through three layers:

1. Action: what was done
2. Effect: what changed in workflow/system
3. Employer value: what pain/risk/cost was reduced

Avoid stopping at layer 1.

## Fast quality checks

Reject or rewrite bullets that only do one of these:

- list tools (`Used X`, `Worked with Y`)
- list participation (`Participated in...`)
- list responsibility without effect (`Responsible for...`)

Keep bullets that clearly show:

- reduced manual work
- improved reliability/visibility
- made delivery more predictable
- reduced operational risk
- improved cross-team execution

## Rewrite templates

- `Reduced [operational pain] by building [solution], enabling [repeatable business result].`
- `Turned [manual/chaotic step] into [automated/controlled process], improving [speed/accuracy/reliability].`
- `Integrated [tool/system] into [existing flow], so [team] could [outcome] with less [risk/rework].`
- `Translated [business requirement] into [technical implementation], helping [stakeholder] achieve [measurable or defensible outcome].`

## Example transformations

1. Before: `Wrote scripts for data processing.`
2. After: `Automated manual input-data preparation into a reusable pipeline, reducing repeated cleanup work and improving analysis consistency.`

1. Before: `Used X tool to complete tasks.`
2. After: `Integrated X into the existing workflow, turning an ad-hoc step into a controlled process with clearer delivery ownership.`

## JD tailoring loop

1. Extract top priorities from JD.
2. Tag each resume bullet to a priority (or mark gap).
3. Rewrite top bullets for highest-priority tags first.
4. Reorder experience bullets so strongest JD-matching evidence appears first.
5. Keep claims truthful and interview-defensible.
